# Screenshot Usage

This file describes how to use the three screenshots you already have.

## Recommended Order

### 1. Main Product Shot

Use the screenshot that shows:

- a live AI page on the left
- the SkillClip workspace on the right

Why:

- it is the strongest product shot
- it shows this is a real browser extension
- it explains the product faster than text

Best caption:

`Compile AI chats into reusable skills beside the conversation itself.`

## 2. Local Assets / Workspace Shot

Use the screenshot that shows:

- current status
- local assets
- workspace management

Why:

- it shows SkillClip is not only an insertion tool
- it communicates local-first asset management

Best caption:

`Review raw captures, reusable skills, variants, and local assets in one workspace.`

## 3. Model Configuration Shot

Use the screenshot that shows:

- validation mode
- provider
- API base URL
- model
- key

Why:

- it explains model-assisted compilation
- it helps technical users understand extensibility

Best caption:

`Plug in your own model for prompt optimization, skill compilation, and checks.`

## Best Usage For GitHub README

Use this order:

1. product shot
2. workspace shot
3. model configuration shot

Recommended file names:

- `hero-workspace.png`
- `local-assets.png`
- `model-config.png`

## Best Usage For X

Use this order:

1. product shot
2. workspace shot
3. model configuration shot

And in the post text, focus mostly on the first two.

For the third screenshot, keep the wording short:

`You can also plug in your own model for compilation and validation.`

## Important Note

The third screenshot is useful, but it is not a growth screenshot.

That means:

- good for technical context
- not ideal as the hero visual

So always keep the first screenshot first.
